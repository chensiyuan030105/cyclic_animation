# cyclic_animation

> 目标：把“用 DMD 做 cyclic animation”这件事尽可能发散地想清楚，先铺满想法，再收敛到可执行路线。

---

## 0. 一句话定位

如果把循环动画看成“在某个低维流形上的近周期动力系统”，那么 DMD 系列方法就是：

- 用可解释的线性/近线性动力学来拟合这个系统；
- 用谱（特征值/模态）直接读出频率、衰减、相位；
- 再叠加接触、约束、控制，把“能循环”变成“可用的循环动画”。

---

## 1. 为什么是 DMD（而不是直接上大模型）

### 1.1 DMD 的独特优势

- 频率与稳定性可解释：每个模态都有振荡频率和衰减率。
- 长时 rollout 诊断清晰：爆炸/衰减/漂移都能在谱上看到。
- 数据效率高：小数据也能做 baseline。
- 与物理系统兼容：和线性化动力学、Koopman 思路天然相容。
- 便于做“控制版”：DMDc、Koopman with control 可平滑接入。

### 1.2 DMD 的天然短板

- 强非线性场景（剧烈接触、大形变）会掉精度。
- 单一线性模型难覆盖多风格、多速度、多技能切换。
- 仅靠重建误差容易“看起来像，但踩地不对”。
- 需要良好的状态表示；原始高维顶点/骨骼直接上会难。

### 1.3 结论

把 DMD 当作“动力学骨架”，而不是“一把梭的最终模型”。

---

## 2. 先明确“循环动画”到底要优化什么

### 2.1 可视目标

- 周期闭合：`x(t+P) ≈ x(t)`。
- 相位稳定：长时间不会越跑越偏。
- 节奏可控：可以改快慢不炸。
- 切换自然：速度、风格、动作切换不卡顿。
- 接触可信：脚不滑、布不穿、碰撞不炸。

### 2.2 可量化目标（建议）

- 周期闭合误差（位置/速度两套）。
- 长时漂移曲线（100 周期、500 周期）。
- 频谱一致性（主频、谐波漂移）。
- 接触违约率（foot skate、penetration）。
- 感知指标（可选）：关键点轨迹平滑度、观感打分。

---

## 3. 数据表示：成败关键

### 3.1 角色动画状态（骨架）

- 基础：关节角 + 角速度 + 根节点速度。
- 进阶：旋转用 6D rotation / quaternion，避免欧拉角跳变。
- 归一化：去全局平移、统一朝向、单位尺度。
- 相位标签：可人工给，也可从主频自动估计。

### 3.2 布料/可变形状态

- 不建议直接全顶点 DMD（维度太高 + 噪声敏感）。
- 推荐先降维：
  - PCA/SVD（最直接）；
  - autoencoder latent（更强但引入非线性）。
- 状态建议包含：位移 + 速度（必要时加应变统计量）。

### 3.3 混合状态（角色 + 布料）

- 方案 A：分开建模再耦合（稳健）。
- 方案 B：联合 latent 后统一 DMD（表达力强但更难调）。

## 4. DMD 家族“武器库”怎么选

### 4.1 经典 DMD

- `X' ≈ A X`，最简单 baseline。
- 作用：快速判断“线性近似是否可行”。

### 4.2 Hankel/Delay DMD（强烈推荐）

- 把时间延迟栈进状态，补偿非马尔可夫性。
- 通常比经典 DMD 稳定得多，特别适合循环信号。

### 4.3 EDMD / Kernel DMD

- 用显式或隐式特征映射提升非线性表达。
- 风险：特征太多会过拟合、泛化差。

### 4.4 DMDc（带控制）

- `x_{t+1}=Ax_t+Bu_t`。
- 适合做速度控制、方向控制、style 滑条。

### 4.5 mrDMD（多分辨）

- 分离不同时间尺度，适合含慢漂移 + 快振荡的序列。

### 4.6 在线/递推 DMD

- 实时更新模型，适合交互式系统或数据持续到来。

---

## 5. 循环稳定性：谱约束怎么做

### 5.1 单位圆约束

- 对主循环模态施加 `|λ| ≈ 1`。
- 防止越滚越衰减或越滚越爆炸。

### 5.2 频率锁定

- 把主模态角频率约束在目标周期附近。
- 可以允许小范围可调（支持节奏变化）。

### 5.3 周期一致性损失

- 直接加 `||x_{t+P}-x_t||` 和 `||v_{t+P}-v_t||`。
- 实操中很有效，尤其在训练数据周期不干净时。

### 5.4 模态稀疏化

- 限制有效模态数，减少高频噪声和“抖动模态”。

---

## 6. 接触与约束：DMD 之外必须补的层

### 6.1 角色足底接触

- 识别接触相位；
- 接触段做足底锁定（IK 或投影）；
- 把 root 调整回动量一致。

### 6.2 布料碰撞/自碰撞

- DMD 先给预测轨迹；
- 再做几何投影或物理修正；
- 必要时局部迭代（只修违反约束的区域）。

### 6.3 约束层策略

- 先“低成本投影”再“高成本解算”；
- 只在违约触发时启用重修正（事件驱动）。

---

## 7. 一个推荐的整体架构（从易到难）

### 7.1 Phase 1：可跑 baseline

- PCA -> Delay DMD -> rollout -> 周期误差评估。
- 先把“会循环”跑通。

### 7.2 Phase 2：稳定与可控

- 加谱约束、周期损失、DMDc 控制输入。

### 7.3 Phase 3：可信视觉

- 接触/碰撞后处理；
- 动作切换与风格插值；
- 长时稳定性验证（上百周期）。

### 7.4 Phase 4：论文/产品化

- 系统性 ablation；
- 速度与质量平衡；
- 封装成推理 API / 实时插件。

## 8. 针对当前仓库可直接落地的实验路线

结合当前仓库现状：

- `extern/physical_cyclic_animations` 有简化循环数据生成逻辑（N-body/cloth/deformable）。
- `extern/Codim-IPC` 可产生更重的物理布料轨迹。

### 8.1 最小闭环实验（建议先做）

- 数据：先用 `square5K` 或 `square113` 生成一批轨迹；
- 表示：PCA 64 或 128 维；
- 模型：Delay DMD（窗口 10~30）；
- 评估：闭环误差 + 长时漂移 + 速度曲线稳定性。

### 8.2 增强实验

- 换 DMDc，输入为外力/目标速度；
- 加周期锁频约束；
- 加碰撞开关对比（with/without collision）；
- 比较不同网格分辨率下的泛化。

### 8.3 论文式对比组

- AR baseline（小 MLP/LSTM）；
- 经典 DMD；
- Delay DMD；
- Delay DMD + 谱约束；
- Delay DMD + 谱约束 + 接触修正。

## 9. 你会遇到的典型坑（提前写）

### 9.1 “能重建，不能 rollout”

- 训练窗口短期误差好看，长时直接漂。
- 对策：强制做 long rollout loss / rollout eval。

### 9.2 “频率对了，幅值崩了”

- 特征值模长偏离 1 或隐空间归一化不稳。
- 对策：模长正则 + 归一化策略一致化。

### 9.3 “视觉上周期了，但接触不对”

- 纯动力学没有语义约束。
- 对策：接触标签 + 接触损失 + 接触投影层。

### 9.4 “高分辨率太慢”

- 直接顶点建模算不动。
- 对策：多级表示（coarse-to-fine）+ 局部修正。

### 9.5 “动作切换发飘”

- 单模型覆盖过宽分布。
- 对策：分相位/分风格子模型 + 门控切换。

## 10. 可发散的高级方向（脑洞区）

### 10.1 DMD + Diffusion

- DMD 负责长时骨架；
- diffusion 负责高频细节与视觉纹理。

### 10.2 DMD + 物理引擎内循环

- 每 K 步用 DMD 预测；
- 中间步用便宜修正；
- 节省大规模求解开销。

### 10.3 模态可编辑

- 直接操控某些模态的相位/幅值；
- 实现“频率变快、摆动变大、风格偏硬/软”。

### 10.4 局部 DMD（Spatially Local）

- 分块建模布料区域（领口、裙摆、袖口）；
- 局部耦合，提升复杂服装可控性。

### 10.5 事件驱动 DMD

- 接触前后切换不同线性系统；
- 类似 hybrid dynamical systems。

### 10.6 Koopman latent + DMD readout

- 非线性编码器负责“线性化”；
- latent 中用 DMD 演化；
- 解码器还原高维状态。

## 11. 具体实验参数建议（初稿）

### 11.1 采样与窗口

- `dt` 固定后，先估主周期 `P`；
- Delay 窗口先试 `w = P/4, P/2, P`；
- 训练时至少覆盖 5~10 个周期。

### 11.2 降维

- PCA 保留 95% 能量；
- 或固定 latent 维度 32/64/128 做对比。

### 11.3 正则

- 谱模长正则 `|| |λ|-1 ||`；
- 周期闭环损失 `L_cycle`；
- 速度平滑 `L_vel`；
- 接触约束 `L_contact`（若有标签）。

### 11.4 评估长度

- 至少 rollout 100 周期；
- 报告 1、10、50、100 周期误差点。

## 13. 如果现在就开工，我的执行顺序

1. 用 `square5K` 先生成干净循环序列（短序列即可）。  
2. 做 PCA + Delay DMD baseline，先产出可视化和指标。

4. 再引入碰撞/接触修正层。  
5. 最后再谈控制（DMDc）与动作切换。

## 15. 一些“先别做”的提醒

- 不要一开始就上超高分辨率 + 全功能约束；
- 不要只看短窗口重建误差；
- 不要忽略相位漂移和接触滑移；
- 不要把所有动作塞进一个线性模型期待奇迹。

---

## 16. 当前共识（暂定）

- DMD 不是终点模型，但非常适合作为循环动画的动力学中枢；
- 真正可用系统 = DMD 动力学 + 约束修正 + 控制接口；
- 先建立“可解释且稳定”的 baseline，再叠加复杂性，成功率最高。
